# 智研舱 Demo 材料包

这套材料用于 `ZYC-12` 的比赛演示。

目标不是展示“上传文档然后聊天”，而是展示一条完整项目主线：

1. 项目空间
2. 证据问答
3. 多文档对比
4. 项目记忆
5. 成果输出
6. 运行轨迹与最小评测

## 建议项目名

`智研舱 Demo 项目`

## 文件说明

- `competition_brief.md`
  - 比赛任务书与评分标准
  - 适合演示项目画像、要求抽取、证据引用
- `solution_plan.md`
  - 参赛方案说明
  - 适合和任务书做对比、生成答辩材料
- `innovation_brief.md`
  - 创新点答辩备忘
  - 适合回答“创新点是什么”
- `demo_runbook.md`
  - 演示与兜底手册
  - 适合回答“一分钟保住哪些页面”
- `evidence_brief.md`
  - 证据问答速查表
  - 适合演示高频问题的证据定位
- `judge_focus.md`
  - 评委追问清单与风险提醒
  - 适合演示长期记忆与输出页

## 建议导入顺序

1. `competition_brief.md`
2. `solution_plan.md`
3. `innovation_brief.md`
4. `demo_runbook.md`
5. `evidence_brief.md`
6. `judge_focus.md`

## 推荐演示动作

### 1. 项目总览

看项目摘要、资料数量、推荐问题。

### 2. 证据问答

推荐问题：

- 这个项目为什么不是普通聊天式知识库？
- 当前方案覆盖了哪些评分项？
- 评委最可能追问什么风险？

### 3. 对比

推荐对比对：

- `competition_brief.md` vs `solution_plan.md`

重点看：

- 已覆盖要求
- 缺失项
- 风险项
- 需人工确认项

### 4. 记忆

重点看：

- 是否沉淀出稳定事实
- 每条记忆是否带 `source_refs`

### 5. 输出

推荐产物：

- 答辩提纲
- 评委问题卡片
- 差异分析摘要

## 演示成功标志

- 回答能给出来源和引用
- 对比结果是结构化的
- 记忆能看到来源
- 产物能回溯到项目上下文
- 3 分钟内能跑完一条闭环
